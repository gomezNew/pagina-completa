    A

    H1 a H6

    Hay diferentes niveles de títulos, del 1 al 6. Las etiquetas nos permiten indicar la importancia del título y para estructurar el contenido, de esta forma ayudamos a los bots a entender la importancia del contenido.

    Definen la cabecera principal de una página h1 , así como los subencabezados (de h2, h3 al h6) de varias secciones del contenido de una página. Desde la perspectiva de las personas y los motores de búsqueda, estas etiquetas funcionan como guía y resumen del texto.

    Img

    Usamos la etiqueta IMG para mostrar imágenes dentro del contenido. Necesita el atributo src para funcionar, ya que será donde indicaremos desde donde tiene que mostrar la imagen.

    ol y li

    1.-El elemento ol permite definir listas o viñetas ordenadas ("Ordered List"), bien con numeración o alfabéticamente.

    El elemento li del ingles item list o elemento de lista declara cada uno de los elementos de una lista.

    Sus etiquetas son: li y /li (la de cierre es opcional).

    ul

    Es una herramienta esencial para crear listas desordenadas en un documento web.Se utiliza para presentar una serie de elementos relacionados que no tienen un orden específico.

    p

    Etiqueta que nos sirve para agrupar texto dentro de un parágrafo. El propósito es poder hacer el contenido más fácil de leer y organizado.

    span

    La etiqueta span es un elemento contenedor en línea genérico y estilo.

    Las etiquetas span generalmente envuelven secciones de texto con fines de estilo o para agregar atributos a una sección de texto sin crear una nueva línea de contenido.

    Metodologías del Desarrollo Web.

    (PROS)Metodologia de Desarrollo Web.

    A: OOHDM

    1.Actualmente sirve como base para el desarrollo de nuevas propuestas metodológicas para los sistemas de información web.
    2. Es una propuesta basada en el diseño, porque ofrece una serie de ideas que han sido asumidas por diferentes propuestas.
    3. Hace uso también de la orientación a objetos y de un diagrama estandarizado, para poder representar el aspecto de navegación (índice, enlaces y nodos).

    B: RMM

    1. Control y visibilidad completa desde una ubicación central.
    2. Gestión integral TI a través de un único panel de control.
    3. Asistencia escalable.
    4. Flujo de trabajo Óptimo
    5. Proporciona la asistencia necesaria sin pérdida de tiempo

    C: UML

    1. Facilita la comprensión de ideas y sistemas complejos.
    2. Convierte el código complejo en un diagrama visual.
    3. Mantiene a todo el mundo en la misma página.
    4. Permite a los desarrolladores ver la imagen global de un sistema.
    5. Ayuda a los no programadores a entender los procesos y funcionamiento del Software.

    D: WSDM

    1. El definir la misión o alcance claramente permite evitar inconsistencias durante el proceso de desarrollo.
    2. Es un proceso dirigido a la audiencia ya que se enfoca en cumplir las necesidades de distintos grupos de usuarios.
    3. WSDM utiliza OWL y RDF las cuales son recomendaciones del W3C (world wide web consortium).
    4. Permite separar el diseño conceptual del diseño de implementación.

    (CONTRAS)Metodlogia de Desarrollo Web.

    A: OOHDM

    1. Fuera de ámbito un aspecto esencial como lo es el tratamiento de la funcionalidad del sistema.
    2. No ofrece ningún mecanismo para trabajar con múltiples actores.
    3. El diagrama navegacional resulta muy complejos para representar gran variedad.

    B: RMM

    1. No permite realizar consultas a partir de dos entidades.
    2. No considera las consultas a la base de datos para la creación de páginas web.

    C: UML

    1. Gran cantidad de modelos necesarios para definir un sistema.
    2. Escasa documentación del proceso.
    3. Falta de apoyo de parte de un grupo u organización.
    4. No considera a las pruebas dentro del proceso de desarrollo.
    5. No existe una herramienta especifica o recomendada para realizar la implementación.

    D: WSDM

    HTML VS CSS

    HTML:

    1. El HTML se utiliza para definir una estructura de una página Web.
    2. Consta de etiquetas dentro las cuales se encierra un Texto o se muestra.
    3. No existen métodos definidos.
    4. No podemos usar HTML dentro de una hoja de estilos (CSS).
    5. El HTML no se utiliza para la presentación y Visualización.
    6. El HTML tiene comparativamente menos respaldo y apoyo.
    7. HTML es el lenguaje de marcado básico que describe el contenido y la estructura de una página Web.
    8. HTML es toda la estructura básica o para dar forma a una sitio web.

    CSS:

    1. CSS se utiliza para dar estilo a las páginas web mediante el uso de diferentes características de estilo.
    2. Consta de selectores y bloques de declaración.
    3. El CSS puede ser interno o externo en función de las necesidades.
    4. Podemos usar CSS dentro de un documento HTML.
    5. El CSS se utiliza para la representación y la Visualización.
    6. El CSS tiene un respaldo y un apoyo comparativamente mayores.
    7. CSS es la extensión de HTML que modifica el diseño y la visualización de las páginas web.
    8. CSS es la parte decorativa de la cual ayuda a darle una mejor y extensa decoración al sitio web.
